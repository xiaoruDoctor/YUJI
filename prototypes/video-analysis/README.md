# 羽迹视频识别技术原型

## 目标

输入一段连续拍摄的羽毛球视频，验证以下核心能力：

1. 检测场上人物和人体关键点。
2. 检测并跟踪目标场地中的羽毛球。
3. 检测击球事件。
4. 识别从发球到落地的独立回合。
5. 估算每个回合的拍数。
6. 输出 JSON、CSV、标注视频和独立回合片段。

## 当前技术路线

首版不依赖单一模型，采用多信号融合：

```text
时序回合检测模型
        +
目标场地 ROI
        +
TrackNetV3 羽毛球轨迹
        +
YOLOv8 Pose 人体与姿态
        +
击球声音候选
        ↓
回合边界、击球事件、拍数与可信度
```

## 已完成的基线验证

### TrackNetV3

- 未限制场地时，职业视频会误检转播图层和固定亮点。
- 业余多场地视频会误检其他场地和右侧固定亮点。
- 裁出目标场地后，能够在业余视频中捕捉真实羽毛球轨迹。
- 当前轨迹存在较多缺口，需要插值、人体动作和声音补充。

### YOLOv8 Pose

- 在业余目标场地 12 秒样本的全部 360 帧中均检测到人物。
- 单帧最多检测到 4 人，能够为单双打和目标场地区域提供稳定信号。

### 羽毛球回合时序模型

- 使用 `phubinhdang/badminton-video-trimmer` 的预训练时序模型。
- 在业余 20 秒样本上输出 `0.23s—20.01s` 的高置信度比赛区间，分数约 `0.973`。
- 能找到正在比赛的粗区间，但可能把相邻回合合并，需要 TrackNet、人体和音频收紧边界。

### 旧版 60 秒业余单打融合结果（错误基线）

> 以下结果仅用于复现旧算法失败，不再作为正式完整回合。旧算法把“人物进入安静状态且视频窗口未截断”误当成完整性证据，导致球尚未落地就提前切断，也会被路人、相邻球场声音和 TrackNet 丢球污染。

目标场地视频输出 5 个回合候选：

| 回合 | 起止时间 | 估算拍数 | 球体支持击球数 | 综合可信度 | 完整性 |
|-|-|-|-|-|-|
| 1 | 4.47—14.62 秒 | 13 | 8 | 0.81 | 旧版误判 |
| 2 | 17.26—26.98 秒 | 15 | 12 | 0.87 | 旧版误判 |
| 3 | 30.79—34.61 秒 | 7 | 2 | 0.64 | 旧版误判 |
| 4 | 38.50—46.37 秒 | 7 | 5 | 0.86 | 旧版误判 |
| 5 | 55.27—60.00 秒 | 3 | 2 | 0.83 | 视频窗口截断，不完整 |

旧版所谓“首尾帧检查”没有确认真实发球接触帧和落地/下网/死球帧，因此结论撤回。严格人工审计真值保存在 `annotations/amateur60_manual_truth.csv`：前四个候选中，目前只有第 2 个能通过“最后击球后无后续击球 + 双方停止争夺 + 比分更新”的组合证据确认死球；其物理触地帧仍是约值。第 1、3、4 个均标为待确认。

当前拍数仍属于估算值，需要逐回合人工真值复核后才能计算拍数误差。

## 本地网页工作台

网页原型使用 FastAPI 和原生 HTML/CSS/JavaScript。视频和项目数据只保存在本机 `outputs/video-web/projects/`，不会上传云端或进入 GitHub。

启动方式：

```bash
cd /Users/lpx/project_user/YUJI
.venv-video/bin/pip install -r prototypes/video-analysis/requirements.txt
.venv-video/bin/python prototypes/video-analysis/run_web.py
```

浏览器打开：

```text
http://127.0.0.1:8765
```

网页流程：

1. 上传本地视频并等待人物、羽毛球、击球和粗候选分析。
2. 在标记视频中逐帧定位发球接触和落地、下网或死球时间。
3. 填写结束证据并确认；未填写死球时间和证据时不能导出。
4. 每个严格回合同时生成标记版和原画质无标记版。

当前一次只处理一个视频。TrackNet 与 YOLO 权重默认从 `.video-research/badminton-pipeline-repro/weights/` 读取，也可以通过 `YUJI_TRACKNET_WEIGHT`、`YUJI_YOLO_WEIGHT` 环境变量覆盖。

模型权重、测试视频和输出均保存在被 Git 忽略的本地目录中。

## 当前限制

- 不同视频需要先指定目标场地 ROI；自动场地选择后续实现。
- 现成 TrackNet 权重对转播图层、灯光和多场地背景有明显域偏移。
- 普通球馆音频包含其他场地击球声，不能只靠声音计拍。
- 回合时序模型负责粗定位，不能单独满足精确发球到落地切片。
- 当前结果属于技术验证，不是产品准确率承诺。
- 当前自动流程只生成回合候选，不再自动声称“完整”。正式完整回合必须具有球轨迹终止、可见落点/下网、无后续击球、比分或下一发球等组合证据；球员停止跑动只能作为辅助证据。
- `shot_count_status` 必须明确区分 `精确拍数`、`估算拍数`、`无法计拍`。本轮四个候选因轨迹缺失和声音串场，全部标为 `无法计拍`，旧 `stroke_estimate` 只保留作错误基线对照。

## 严格版验收标准

- `serve_time` 对准首次有效发球接触，不用发球准备或人物开始移动代替。
- `landing_time` 对准可确认落地、下网或死球事件；无法确认时留空并标 `pending_confirmation`。
- TrackNet 暂时丢球、击球间隔或球员停止跑动都不得单独触发正式结束。
- 调试视频中直接检测为黄色小光圈，插值为紫色小光圈；击球瞬间只轻微放大，并绘制细轨迹和方向箭头。异常大跨度跳点不会连接成误导性长线。
- 轨迹丢失时不外推、不继续绘制误导性点；主画面不再放置遮挡球场的右上角局部放大窗。
- 严格切片目录默认只导出 `is_complete_rally=true` 的回合，并从原始 1920×1080 视频生成高质量 H.264 MP4。
