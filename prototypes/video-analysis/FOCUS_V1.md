# 指定球员与羽毛球轨迹 V1

## 验证目标

输入一段固定转播机位视频，通过首帧点击坐标指定一名球员，输出：

- 仅包含目标球员框的标记视频。
- 羽毛球直接检测点与插值点。
- 羽毛球最近若干帧的短轨迹。
- 保留原视频音频的 H.264 MP4。

本阶段不判断回合、球路类型、比分和击球动作。

## 运行方式

先生成逐帧人物结果和融合后的羽毛球结果，再运行：

```bash
.venv-video/bin/python prototypes/video-analysis/scripts/render_focus_video.py \
  --video 输入视频.mp4 \
  --people-csv 人物结果.csv \
  --ball-csv 羽毛球结果.csv \
  --selection-frame 0 \
  --selection-x 1100 \
  --selection-y 580 \
  --output 聚焦标记视频.mp4
```

`selection-x` 和 `selection-y` 是指定帧中目标球员身体范围内的一次点击坐标。若点击位置没有落在人物框内，程序选择距离点击位置最近的人物。

## 首个职业比赛基线

样本：`outputs/video-prototype/samples/pro_single_20s.mp4`

- 总帧数：600。
- 目标人物同一 Track ID 有框帧：379。
- 目标人物同一 Track ID 覆盖率：63.2%。
- 羽毛球直接检测帧：204。
- 羽毛球插值帧：29。
- 羽毛球轨迹覆盖率：38.8%。

这组数据只作为首轮改进基线。转播切镜、短暂遮挡和 Track ID 切换仍会造成目标框中断；球轨迹也不能视为完整真实飞行轨迹。

## 验收标签

人工检查时只记录四类错误：

- `跟丢`：目标球员可见但没有人物框。
- `串人`：人物框跟到其他人。
- `漏球`：羽毛球清晰可见但没有球点。
- `误标`：球点落在白线、灯光、广告牌或其他非羽毛球目标上。
