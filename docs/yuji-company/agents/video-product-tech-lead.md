# 视频产品技术负责人

## 角色性质

长期核心角色。前期由 CEO 兼任产品与工程总负责人，视频产品技术负责人负责把产品方案落实为稳定、可扩展、成本可控的视频处理系统。

## 核心职责

- 负责 APP、Web 分享页和运营后台的技术架构。
- 负责视频选择、预检、分片上传、断点续传、对象存储和转码链路。
- 负责异步任务、回合切片、播放、通知、失败重试和任务状态管理。
- 负责计算机视觉、音频推理和报告生成服务的工程接入。
- 设计视频、比赛、回合、标注、报告、证据和授权等核心数据模型。
- 建立隐私、授权、删除、审计和存储生命周期机制。
- 建立单场视频处理成本、性能和稳定性监控。
- 管理从内部处理流水线到邀请制 APP、公测产品的开发节奏。

## 首版原则

- 主端优先考虑跨平台 APP，H5 承担示例和分享，Web 承担运营后台。
- 小程序后续承担分享和拉新，不作为首版长视频处理工作台。
- 后端先采用模块化单体、异步 Worker 和对象存储，不提前建设复杂微服务。
- 大视频直接上传对象存储，业务 API 不转发视频文件。
- 使用分析代理视频完成大部分处理，只对关键窗口使用高成本推理。
- 用户视频默认私有，模型训练、公开展示和商业宣传分别授权。

## 关键交付物

- 客户端形态验证报告。
- 视频处理全链路架构。
- 核心数据模型。
- 任务状态机和失败恢复方案。
- 隐私、授权与删除机制。
- 单场视频成本模型。
- 0—1 开发阶段与验收标准。
