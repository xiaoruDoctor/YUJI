# YUJI 运营素材库

本目录用于沉淀每天自动化、CEO 想法、外部观察、评论私信和发布复盘中产生的可复用内容资产。

素材库不是待发布笔记，也不是日报归档。它负责保存“今天不一定用，但以后可能会用”的素材、灵感和产品信号，避免好东西散在每日文件里。

## 目录文件

| 文件 | 用途 |
|-|-|
| `materials.md` | 原始素材池：外部观察、用户表达、链接、评论私信、真实场景 |
| `topics.md` | 选题池：可转成小红书笔记的内容方向、标题和切入角度 |
| `product-signals.md` | 产品信号池：可能转成 Player ID、羽迹时刻、战损档案等 MVP 的线索 |

## 与每周研究会的关系

每周“羽毛球情绪价值雷达”研究会结束后，用户研究专家需要把结论拆分写入本素材库：

- 真实用户表达和社区观察进入 `materials.md`。
- 可转成小红书内容的方向进入 `topics.md`。
- 可低成本验证的 MVP 假设进入 `product-signals.md`。

研究会主文档位于 `../research/badminton-emotion-value-radar.md`，会议模板位于 `../research/weekly-template.md`。

## 入库原则

- 每天自动化产出的素材，主稿用到的可以标记为“已使用”，没用到的也要入库。
- CEO 临时想到的点、球友聊天、评论区故事、私信反馈，都可以入库。
- 入库时保留来源、来源说明、来源链接、日期、情绪主题、标签和可复用方式。
- 不把未验证灵感直接写成品牌结论；先放入库，后续用内容数据验证。
- 不收录会把 YUJI 带偏成装备测评、技术教学、AI 炫技或低价定制的素材。

## 来源记录要求

- `materials.md` 至少保留 `来源`、`来源说明`、`来源链接`，不能只留一个平台名。
- `topics.md` 作为灵感库 / 选题池，也要保留来源细节，至少写清 `来源素材`、`来源说明`、`来源链接`，方便后续回查和二次验证。
- 如果是搜索摘要、转载页、站外引用或抓取不完整，必须在 `来源说明` 里明确写出来，不把弱证据包装成完整原帖。
- 如果一条选题来自多条素材，写清主来源和辅来源，链接可以并列保留。

## 每日使用流程

1. 09:00 或 18:00 自动化产出后，把可复用素材写入 `materials.md`。
2. 把当日未发布但有潜力的内容方向写入 `topics.md`。
3. 把可能形成 MVP 的用户需求、私信或评论写入 `product-signals.md`。
4. 写小红书笔记前，优先从近期自动化和本素材库里各选 1-2 个点交叉判断。
5. 发布后，根据评论和数据更新相关条目的状态。
6. 每周研究会后，把跨平台研究结论按素材、选题和产品信号拆分入库。

## 状态约定

- `未使用`：已入库，尚未转成笔记或产品动作。
- `优先储备`：CEO 明确点名保留，后续优先写成笔记或优先进入草稿。
- `已用于笔记`：已经进入草稿、待发布或已发布笔记。
- `待验证`：值得用内容或私信进一步验证。
- `验证中`：已经发布或开始收集反馈。
- `已沉淀`：已经写入 Skill、产品方向或稳定内容栏目。
- `暂缓`：暂时不符合阶段，先不使用。
