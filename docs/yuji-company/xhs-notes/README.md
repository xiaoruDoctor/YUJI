# YUJI 小红书发布内容库

本目录用于沉淀 YUJI 小红书内容资产，记录从草稿、待发布、已发布到复盘反馈的完整过程。

## 目录说明

| 目录 | 用途 |
|-|-|
| `drafts/` | 草稿、原始情绪、未定稿选题 |
| `ready-to-publish/` | 已整理完成、可发布的笔记 |
| `published/` | 已发布笔记归档，包含发布时间和发布链接 |
| `feedback/` | 发布后的数据、评论、私信和复盘 |

## 发布节奏

- [小红书冷启动发布计划](./publishing-plan.md)：记录 YUJI 小红书从账号配置完成后的首月发布频率、发布时间测试和复盘指标。
- [小红书发布前检查清单](./publishing-checklist.md)：记录发布类型、封面、标签、后台声明、组件、定时和首评规则。

## 单篇笔记建议结构

- 基本信息：标题、状态、计划发布时间、内容类型。
- 创作来源：CEO 原始想法、真实经历、情绪关键词。
- 发布内容：标题备选、正文、封面文案、配图建议、标签。
- 生图提示词：每条配图建议对应一条可直接给 ChatGPT 使用的细节提示词。
- 评论互动：置顶评论、评论区引导话术。
- 品牌校验：是否符合 YUJI 当前阶段和表达边界。
- 发布后台配置：发布类型、内容类型声明、标签取舍、可见性、组件、定时发布和首评提醒。
- 发布后观察：收藏、评论、转发、私信和产品信号。

## 内容原则

- 第一阶段先建立羽毛球人的情绪审美，不急着卖产品。
- 前台表达不讲 AI 和工程化，AI 只是幕后能力。
- 写具体场景，不写抽象口号。
- 先让球友觉得“你懂我”，再让他们觉得“我也想拥有一个”。
- 每篇正式发布内容都要能回到 YUJI 的核心：让每一次挥拍，都留下热爱的痕迹。

## 与运营素材库的关系

写笔记前优先检查：

1. 当天 `daily/` 自动化产物。
2. `docs/yuji-company/content-library/materials.md` 中近期未使用素材。
3. `docs/yuji-company/content-library/topics.md` 中 `未使用` 或 `待验证` 选题。
4. `docs/yuji-company/content-library/product-signals.md` 中适合轻量试探的产品信号。

每篇笔记沉淀后，应回写：

- 使用了哪些素材。
- 哪些备选标题或角度没有使用，但值得放入选题池。
- 是否产生新的产品信号。
- 发布后是否需要更新素材、选题或产品信号状态。
