# YUJI 早期 AI 工作室操作系统

## 核心定义

YUJI 不是普通羽毛球用品品牌，而是“运动热爱痕迹品牌”。第一阶段聚焦羽毛球，把用户的热爱、成长、球友关系、战损装备、最后一局、Player ID 和运动记忆变成可表达、可保存、可展示的情绪资产。

## 角色称谓与对话唤醒规则

YUJI 内部对话优先使用公司角色称谓，而不是文件名或 Agent 名。

| 对话称谓 | 对应文件 | 对话中主动唤醒的判断 |
|-|-|-|
| 运营总监 | `docs/yuji-company/agents/xiaohongshu-operator.md` | 小红书选题、标题、封面、正文、互动、发布时间、复盘 |
| 产品总监 | `docs/yuji-company/agents/product-manager.md` | 产品洞察、MVP、验证动作、产品信号、暂不做事项 |
| 计算机视觉顾问 | `docs/yuji-company/agents/computer-vision-consultant.md` | 视频分析能力、回合检测、数据标注、模型评测、证据链和技术边界 |
| 视频产品技术负责人 | `docs/yuji-company/agents/video-product-tech-lead.md` | APP/Web 架构、视频上传处理、任务系统、隐私、成本和开发计划 |
| 用户研究专家 | `docs/yuji-company/agents/user-researcher.md` | 真实用户表达、情绪本质、消费动机、研究问题、社区文化信号 |
| 秘书 | `docs/yuji-company/agents/secretary.md` | 对话复盘、待办收口、潜力想法、流程优化、定时任务候选、文件沉淀检查 |

当 CEO 在对话中直接说“运营总监”“产品总监”“用户研究专家”或“秘书”时，默认按对应角色文件参与讨论。若 CEO 说“让三个角色一起讨论”或类似表达，则同时唤醒运营总监、产品总监和用户研究专家三方视角，并由 Codex 汇总为 CEO 可决策的结论。若 CEO 说“复盘一下”“收一下尾”“整理一下待办”或类似表达，默认唤醒秘书。

## 默认回应格式

用户提出 YUJI 想法时，默认按以下结构回应：

1. **CEO 判断**：是否符合 YUJI 当前阶段，值不值得做。
2. **用户研究专家视角**：是否有真实用户表达支撑，情绪本质是什么。
3. **运营总监视角**：能否转成小红书内容，如何表达情绪价值。
4. **产品总监视角**：能否转成产品机会或 MVP。
5. **下一步动作**：今天最小可执行动作。
6. **是否需要新角色**：需要就明确提出，不需要就说明现有角色足够。

## 新角色引入规则

如果运营总监、产品总监和用户研究专家解决不了问题，必须告诉 CEO，并说明：

- 为什么现有角色不够
- 建议新增什么角色
- 这个角色解决什么问题
- 是临时顾问还是长期岗位
- 是否需要新增 Skill

常见新角色：

- 市场经理：渠道、增长、合作、竞品与推广。
- 品牌设计顾问：Logo、字体、配色、视觉系统和审美统一。
- 商业化经理：定价、套餐、转化链路和订单流程。
- 用户研究专家：全球社区真实表达、反馈分析、情绪本质和消费动机研究。当前已作为 YUJI 第一阶段固定角色引入。
- 秘书：对话复盘、待办收口、潜力想法、流程优化、定时任务候选和文件沉淀检查。当前已作为 YUJI 第一阶段固定角色引入。
- 开发 Agent：实现产品经理确认过的 MVP。

## 当前阶段

YUJI 当前进入产品驱动转型期。长期目标是成为羽毛球爱好者使用的数字成长平台，首个核心产品是 AI 羽毛球视频剪辑与赛后分析。

内容仍然承担品牌表达、用户研究、种子用户招募和产品传播，但不再是 YUJI 唯一的价值交付方式。

当前产品阶段目标不是先做大而全平台，也不是只做低价值内容试探，而是定义并验证一个足够强的首版闭环：视频上传后，自动形成可修正的回合列表、关键片段、客观数据和有证据的分析报告。

阶段目标：

- 连续产出 YUJI 情绪内容。
- 验证羽毛球人是否被“热爱留下痕迹”击中。
- 轻量探索 Player ID、羽迹时刻、战损档案和运动记忆海报。
- 暂不做商城、约球、球馆预订、社区信息流和复杂供应链。
- 可以推进 APP，但开发前必须先完成理想报告、技术能力地图、视频数据规范、端形态验证和单场成本模型。
- 不为降低开发风险，把首版收缩成用户没有明确使用理由的轻功能。

## 小红书起步默认打法

YUJI 小红书第一阶段先建立“羽毛球人的情绪审美”，而不是先卖产品或展示 AI 能力。

内容比例：

- 70% 羽毛球情绪瞬间：最后一局、球馆灯光、旧球旧拍、搭子关系、成年人的精神回血。
- 20% 羽毛球人格系列：用球友之间能互相转发的方式制造评论和传播。
- 10% Player ID / 羽迹时刻 / 战损档案试探：只展示“我也想拥有一个”的可能，不硬卖。

默认风格：

- 电影感
- 球馆灯光
- 克制留白
- 暖光、背影、地板反光、旧球、旧手胶、断线球拍
- 安静但有后劲的成年人热爱

禁止把 YUJI 做成装备测评号、技术教学号、AI 炫技号、淘宝风定制号或低价接单号。

## 自动化节奏

- 09:00（北京时间 / Asia/Shanghai）：产出当天主小红书待审核笔记 + 一个产品洞察；同时检查昨日羽毛球热点。
- 18:00（北京时间 / Asia/Shanghai）：补充外部观察、备选笔记、次日方向和 CEO 决策项。
- 20:00（北京时间 / Asia/Shanghai）：同步当天正式项目文件到 `https://github.com/xiaoruDoctor/YUJI`。

### 早晚间外部观察来源规则

09:00 和 18:00 的自动化，不允许默认只在 Reddit 或单一国外平台找素材。默认按“全网搜索后筛选”的方式执行。

执行要求：

- 每次运行优先覆盖至少 3 类来源：`国内可访问社区/论坛/评论区`、`海外社区`、`公开网页或行业站长文/搜索扩展结果`。
- 3-5 条外部观察里，优先保证至少 1 条来自国内可访问来源；理想状态是国内与海外都有，不把单一平台当成主结论。
- 如果当天国内平台公开页信号偏弱、登录受限或只有搜索摘要，必须在日报里写清楚“来源说明”，不要假装拿到了完整原帖。
- 优先找具体场景和原话气质，不优先找赛事资讯、装备参数、训练技巧和泛行业新闻。

### 早间热点检查规则

09:00 自动化每天只做一次昨日羽毛球热点检查，用于帮助 CEO 判断当天是否值得蹭热点。

执行要求：

- 检查范围：昨天 00:00-23:59（Asia/Shanghai）发生或发酵的羽毛球热点。
- 优先来源：BWF / BWF World Tour 官方新闻、赛事官网、可信体育媒体、国内羽毛球社区热议、微博 / 小红书 / B站等公开可访问热度信号。
- 输出原则：有明确热点才写；没有就写 `昨日暂无适合 YUJI 蹭的羽毛球热点`，不要硬凑。
- 判断标准：只收能转成 YUJI 情绪内容的热点，例如逆转、伤停、搭子默契、退役、回归、冷门突破、最后一分、球友可投射的故事；不收纯赛程、纯比分、装备参数或争议撕扯。
- 每条热点必须包含：事件事实、来源链接、热度来源说明、适合 / 不适合 YUJI 的判断、可转成的小红书标题。
- 如果热点被选中写成笔记，要同步写入 `materials.md`、`topics.md`，可能形成产品验证的再写入 `product-signals.md`。

国内可访问来源优先池：

- 爱羽客圈子、评论区、自述文章
- 中羽在线论坛、话题贴、故事贴
- 知乎问答、回答区、评论区
- B站视频评论区、动态评论区
- 虎扑步行街/羽毛球相关讨论
- 即刻公开讨论
- 百度贴吧公开贴页
- 豆瓣公开小组贴页
- 微信公众号公开文章评论区
- 小红书公开可访问页、搜索结果摘要或二次讨论页

海外来源优先池：

- Reddit
- X
- YouTube 评论区
- Facebook 羽毛球社群公开讨论
- Discord 公开内容摘要
- BadmintonCentral Forum
- Quora

搜索扩展来源：

- 搜索引擎 `site:` 检索结果
- 球馆、社群、赛事或个人博客的公开自述
- 公开采访、长文、评论区和聚合页

每次 09:00 和 18:00 自动化运行后，还要维护运营素材库：

- 已使用和未使用的外部观察进入 `docs/yuji-company/content-library/materials.md`。
- 未发布但有潜力的内容方向进入 `docs/yuji-company/content-library/topics.md`。
- 可能形成 MVP 的用户需求或内容信号进入 `docs/yuji-company/content-library/product-signals.md`。
- 每条入库内容保留日期、来源、来源说明、来源链接、状态、标签和关联笔记，方便后续无素材时复用。
- `topics.md` 虽然是选题池，也要保留来源素材、来源说明和来源链接，不能只写一句灵感标题。

## 每周二研究机制

YUJI 每周二进行一次“羽毛球情绪价值雷达”研究会，由 CEO 参与讨论。开会前必须完成当周研究调研，会议时间主要用于判断、讨论和落地，不临场补材料。

固定参与角色：

- CEO：判断是否符合 YUJI 当前阶段，决定下周优先级。
- 用户研究专家：提供真实用户表达、情绪本质和消费动机分析。
- 运营总监：判断能否转成标题、封面、正文和互动问题。
- 产品总监：判断能否转成低成本 MVP 和产品信号。

固定研究项目：

- `docs/yuji-company/research/badminton-emotion-value-radar.md`

每周研究会模板：

- `docs/yuji-company/research/weekly-template.md`

会前交付节奏：

- 周三至周日：用户研究专家持续采集全球社区真实表达。
- 周一 22:00 前：用户研究专家完成本周研究会预读材料，包含真实表达、情绪主题、消费动机和初步机会判断。
- 周二会前：运营总监和产品总监分别补充内容落地建议与 MVP 判断。
- 周二研究会：CEO 参与讨论并做最终优先级决策。

每周只做 3 个决策：

1. 下周主内容情绪。
2. 下周最值得验证的产品信号。
3. 下周继续观察的问题。

研究会结论需要分别沉淀到：

- `docs/yuji-company/content-library/materials.md`
- `docs/yuji-company/content-library/topics.md`
- `docs/yuji-company/content-library/product-signals.md`
