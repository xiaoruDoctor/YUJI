# 小羽视觉资产工作流

本工作流用于生成、审核、保存和使用小羽的图片与视频素材。小羽是 YUJI 的固定前台视觉角色，但不是每条内容都必须出现。

## 1. 生成首张正脸基础图

1. 打开 `prompts.md`。
2. 复制“提示词 1：小羽正脸基础图”。
3. 生成多张候选图。
4. CEO 选定最符合 YUJI 气质的一张作为小羽正脸基础图。
5. 保存为 `docs/yuji-company/visual-system/xiaoyu/assets/base-reference/xy-base-01-front.jpg`。
6. 在 `assets-index.md` 中把 `XY-BASE-01` 状态改为“可用”，并记录审核结论。

## 2. 生成后续角度图

1. 上传已选定的 `xy-base-01-front.jpg` 作为参考图。
2. 依次复制“提示词 2-6”生成不同角度和场景。
3. 每张图必须保持同一个人的脸型、五官、发色和气质。
4. 图片通过 `qa-checklist.md` 后，保存到 `assets/base-reference/`。
5. 更新 `assets-index.md` 的状态和备注。

## 3. 小红书使用

写小红书笔记时，配图建议需要加一道判断：

> 这个位置如果可以有人出现，先想一遍：是不是应该让 YUJI 的女生球友小羽出现？

适合小羽出镜时，配图建议要写清：

- 使用哪一张小羽基础参考图。
- 小羽在画面里的动作。
- 画面承载的羽毛球情绪。
- 需要避开的风险，例如擦边、写真、低胸、木地板、篮球馆。

不用小羽时，也要简单说明原因，例如“这篇更适合旧球和旧拍静物，不让人物抢走情绪”。

## 4. 视频使用

视频中小羽可以承担：

- 下班后走进球馆。
- 场边握拍看球。
- 系鞋带、整理拍包、喝水。
- 接发前半蹲准备。
- 最后一局后收包离开。
- 作为羽迹卡或 Player ID 样稿里的示例人物。

视频不要做成虚拟偶像表演、技术教学口播、擦边运动写真或 AI 展示视频。

## 5. 复盘与更新

每次小羽出现在正式内容后，在 `assets-index.md` 的使用记录中补充：

- 日期。
- 使用了哪张图。
- 使用在哪篇小红书笔记、视频或品牌物料。
- 数据和评论反馈。
- 是否继续强化这个角度，或是否需要生成新角度。

如果小羽相关内容开始吸引偏离 YUJI 的流量，需要降低使用频率，并回到静物、球馆、旧物、搭子关系和羽迹卡样稿等视觉线。
