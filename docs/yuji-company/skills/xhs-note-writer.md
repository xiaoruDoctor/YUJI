# Skill：小红书笔记生成

## 适用场景

把羽毛球情绪观察转成 YUJI 风格的小红书待审核笔记。

本 Skill 同时作为小红书笔记总控：当 CEO 只说“生成一篇小红书笔记”“发小红书”“这张图怎么发”但没有明确类型时，先判断内容类型，再选择对应写法或调用更具体的 Skill。

## 起步定位

小红书第一阶段先建立“羽毛球人的情绪审美”，不要做成产品号、装备号、教学号、AI 做图号或低价定制号。

YUJI 在小红书上的角色是：羽毛球人愿意投射热爱、记忆、搭子关系和成长痕迹的地方。

## 内容比例

第一阶段按以下比例规划内容：

- 70% 羽毛球情绪瞬间：负责收藏、共鸣和品牌气质。
- 20% 羽毛球人格系列：负责评论、转发和社交传播。
- 10% Player ID / 羽迹时刻 / 战损档案试探：负责验证“我也想拥有一个”的产品信号。

不要反过来。产品露出和 AI 作品只能服务情绪表达，不能成为内容主角。

轻图文可以作为补充发布形态，用于提高账号亲近感、建立小羽记忆点和低成本测试一句情绪，但不能替代主情绪图文，也不能把账号带成穿搭号或角色美图号。

## 笔记类型路由

当 CEO 想生成小红书笔记时，先判断属于哪一类：

| 类型 | 触发词 / 场景 | 使用方式 |
|-|-|-|
| 小羽轻图文 | “轻图文”“今日份小羽”“小羽 OOTD”“一张图配一两句话”“小羽日常” | 使用 `docs/yuji-company/skills/xhs-light-post.md` |
| 重情绪图文 | “完整笔记”“重一点”“讲一个情绪故事”“主笔记” | 使用本 Skill 的完整笔记流程 |
| 用户故事 | 球友投稿、评论、私信、真实经历改写 | 使用本 Skill 的用户原话与截图规则，必要时后续拆独立 Skill |
| 小羽日常 | 小羽第一人称、公司成员、女生球友日常 | 先判断是否轻图文；需要完整叙事时使用本 Skill |
| 产品试探 | 羽迹卡、Player ID、战损档案、样稿验证 | 使用本 Skill，并同步参考 `product-insight.md` |

如果 CEO 已明确类型，直接进入对应流程，不要反复提问。

如果类型不明确，只问一个问题：

```text
这篇想做哪种：轻图文、完整情绪图文、用户故事、小羽日常，还是羽迹卡/Player ID 试探？
```

默认选择原则：

- 已有一张小羽图或参考图，只想配一句情绪：走小羽轻图文。
- 有完整情绪、故事、选题或产品验证目的：走重情绪图文。
- 有真实球友表达：走用户故事。
- 有羽迹卡、Player ID、战损档案：走产品试探。

## 固定产出

以下是完整情绪图文或产品试探笔记的默认产出。若路由到 `xhs-light-post.md`，按小羽轻图文的轻量格式输出。

- 标题 3 个
- 正文 1 篇
- 封面文案 1 条
- 配图建议 2-3 条
- 每条配图建议对应的 ChatGPT 生图提示词
- 小羽出镜判断：如果本篇需要人物承载情绪，必须判断是否由 YUJI 的女生球友小羽出镜；如果不用小羽，要说明原因
- 标签 8-12 个
- 评论区互动话术 2 条
- 给 CEO 发布用的一键复制块：正文、标签、首评
- 发布后台配置建议
- 品牌校验
- 可复用经验判断：本次是否有值得写回 Skill 的新结构、新句型、新禁区或新判断
- 素材库更新判断：本次使用了哪些素材，哪些未使用灵感需要进入选题池

## 单篇发布流程

每篇小红书笔记从选题到发布，按这个顺序推进：

1. 从选题池、素材库或 CEO 新想法中选题，说明为什么适合当前发布日期和阶段。
2. 给出运营判断、标题备选、推荐标题、正文、封面文案、配图建议和生图提示词。
3. 如果 CEO 明确喜欢某个主题或标题，要立刻把它同步到推荐标题、封面文案、后台标题、选题池和发布计划，不保留旧标题作为主表达。
4. CEO 生成图片后，检查地胶、人物组合、文字可读性、画面情绪和叙事顺序；通过后复制到项目素材目录，并在待发布稿里写入配图定稿和视觉审核。
5. CEO 准备发布时，直接给正文、标签、首评的一键复制代码块，不让 CEO 自己从长文档里摘。
6. CEO 设置定时发布后，更新待发布稿状态、实际后台配置、首评状态和发布计划。
7. 正式发布后 24 小时内，再写入 `feedback/`，记录数据、评论、私信和产品信号。

## 内容结构

1. 开头先写一个球友真实会做的小动作、会说的一句话或会遇到的具体场景。
2. 中段写具体画面和球友共鸣，不要先解释洞察。
3. 结尾不能只写品牌宣言。根据当篇场景选择评论、私信、投稿或羽迹卡样稿入口，让读者知道下一步能做什么。
4. 如有产品露出，只占 20%，不要硬卖。优先把产品形态做成“看得见的结果”，例如一张羽迹卡海报，而不是解释品牌使命。

## 正文标点规则

小红书正文采用短句换行风格，正文和给 CEO 的一键复制块正文里，句末默认不加中文句号 `。`。

执行要求：

- 短句换行后不要补句号，靠换行形成节奏。
- 可以保留问号、引号、冒号和必要的顿号、逗号。
- 标题、来源说明、运营判断、发布后台配置等内部说明不受此规则限制。
- 交付前检查 `## 正文` 和 `【正文】` 代码块，确认没有遗留句末 `。`。

## 结尾和羽迹卡入口

“羽迹卡”是第一阶段更适合前台使用的产品名。它不是泛泛的 Player ID，也不是接单定制海报，而是把一个球员、球友、搭子或一场球的故事、事迹、场景和情绪，整理成一张有审美、有记忆点、可保存、可分享的羽毛球故事海报。

使用原则：

- 不要每篇笔记都硬接羽迹卡。只有当文章里有足够具体的人、故事、比分、旧物、搭子关系或难忘场景时，再自然引出。
- 先给读者看见结果，再邀请参与。优先在图文最后一张或补充图里展示一张高质感羽迹卡样稿。
- 热点运动员内容可以先给运动员做一张有逼格的羽迹卡，作为读者理解产品形态的样板；发布前必须校对赛事、比分、日期等事实，不把运动员写成品牌背书。
- 职业球员热点内容如果球员本身有明确识别度，封面和后台标题可以直接出现球员姓名，再搭配具体比赛节点、比分或赛季节点；姓名负责让用户识别和搜索，具体节点负责提供情绪冲突。不要把强识别信息过度抽象成泛泛情绪标题。
- 普通球友内容可以写“我先免费帮 5 个球友做一张羽迹卡”，但要保持品牌气质，不写成低价接单、批量定制或模板海报。
- 羽迹卡入口要具体到用户动作，例如评论一个瞬间、私信一句暗号、提交球龄/场景/照片/一句话。

可复用结尾句式：

```text
我想把这个瞬间做成一张羽迹卡。
不是战报，也不是荣誉榜。
是一张能把这场球、这个人、这段热爱留下来的海报。
```

```text
如果你也有一场一直记得的球，
可以在评论区留：时间 / 和谁 / 为什么忘不了。
我先免费帮 5 个球友做一张羽迹卡。
```

```text
这篇后面那张图，就是我想做的羽迹卡样子。
如果你也想把自己的某一局留下来，可以私信我：羽迹卡。
```

## 反总结腔规则

小红书正文不能写成“正确地总结了洞察”。如果一个真实打球的人读了没有被带进场景，就算洞察正确，也不是好笔记。

写作时优先使用：

- 小动作：收包时把旧球塞回侧袋、球馆快关灯还在说最后一局、手胶磨黑了还没换。
- 具体感官：灯光、羽毛球地胶反光、喘气声、鞋底摩擦声、拍包、汗印、水喝空了。
- 球友原话：`再来一局？`、`最后一局了`、`这个球还能练两拍`。
- 第一人称或第二人称心里话：`你拍包里有没有过这样一颗球？`

避免使用：

- 先下结论再举例，比如“成年人很多热爱其实没有那么热血”。
- 抽象解释堆叠，比如“情绪价值、身份感、记忆资产、产品机会”。
- 像运营复盘一样的句子，比如“这说明用户对旧物有情感连接”。
- 品牌替用户解释太多，比如一开头就说“YUJI 想记住的是热爱留下的痕迹”。

自检标准：

1. 去掉品牌名后，这篇是否仍像一个真实球友会发的心里话。
2. 前 5 行是否出现具体动作或画面，而不是抽象判断。
3. 正文里是否至少有 2 个羽毛球人熟悉的细节。
4. 读者是否能在评论区自然接一句“我也有过”。

## 用户原话与截图规则

真实用户有感情的话，可以成为小红书笔记的强共情素材，但不能默认直接搬运截图。

优先级：

1. **授权截图**：用户明确同意后，可使用原评论或私信截图，并打码头像、昵称、ID、地区、群名等身份信息。
2. **匿名摘录**：未获得授权时，不放原截图；可以改写成“根据球友讨论整理”的情绪摘录卡。
3. **聚合表达**：多条用户讨论提炼成一句球友心里话，不冒充某个真实用户原话。

截图或摘录使用要求：

- 不把用户原话放成品牌背书或广告证言。
- 不出现可识别个人身份的信息。
- 不截取对方头像、昵称、主页、定位、群聊名称和其他平台水印。
- 不使用带有争议、攻击、隐私、医疗、家庭困境等敏感信息的原截图。
- 如果来自其他平台，正文可写“看到一位球友说过”或“有球友说”，不要制造对方授权代言的感觉。
- 如使用授权截图，应在内部发布稿中记录授权方式和日期，便于后续追溯。

适合呈现的位置：

- 第 2 或第 3 张配图，用作“球友留言 / 情绪摘录卡”。
- 不建议作为首图封面，首图仍优先使用 YUJI 的球馆情绪视觉。

示例标注：

- `来自一位球友的授权留言，已隐去身份信息。`
- `根据公开球友讨论整理，非原文截图。`

## 选题池规则

每次写完小红书笔记后，必须更新或检查 `docs/yuji-company/content-library/topics.md`。

需要写入选题池的内容：

- 当天自动化产出但没有进入主稿的备选标题、切入角度和情绪主题。
- 已经成稿但未来还能复写成系列的主题。
- 评论区、私信或 CEO 想法里出现的可扩展选题。
- 能帮助下次无素材时快速启动的“半成型选题”。

每条选题至少记录：

- 日期
- 状态
- 内容类型
- 来源素材
- 推荐标题
- 核心情绪
- 开头切入
- 评论互动
- 可关联产品信号
- 标签
- 关联笔记

写稿前优先检查：

1. 当天自动化产物。
2. `content-library/materials.md` 中近期未使用素材。
3. `content-library/topics.md` 中 `未使用` 或 `待验证` 选题。

如果选题被写成草稿、待发布或已发布笔记，要回到选题池把状态改为 `已用于笔记`，并补充关联笔记路径。

## 小羽配图建议规则

小羽是 YUJI 工作室的固定前台视觉角色，是 YUJI 的虚拟女生球友。写小红书配图建议时，如果画面需要人物，优先判断是否使用小羽。

适合小羽出镜的笔记：

- 下班后去球馆。
- 羽毛球是情绪出口。
- 场边休息、刚打完一局还想继续。
- 女生球友视角。
- 需要小红书封面人物停留感。
- 羽迹卡、Player ID、羽迹时刻需要示例人物。

配图建议中如果使用小羽，必须写清：

- 使用哪个小羽基础参考图作为角色参考。
- 使用 `docs/yuji-company/visual-system/xiaoyu/outfit-library.md` 中哪一套服装；不同笔记可以换衣服，但优先最近 14 天不重复。
- 小羽在画面中的动作和情绪。
- 画面如何服务当篇笔记，而不是单独展示角色。
- 如果同一篇笔记连续使用 2-3 张小羽图片，必须锁定同一套服装和道具，提示词里逐项写明上衣颜色、袖长、下装颜色、鞋袜、拍包、发型和主要道具，避免第二张、第三张自动换衣服。
- 需要参考 `docs/yuji-company/visual-system/xiaoyu/prompts.md`。
- 需要按 `docs/yuji-company/visual-system/xiaoyu/qa-checklist.md` 做发布前检查。

不用小羽的合理原因：

- 这篇更适合旧球、旧拍、手胶、空球馆等静物视觉。
- 这篇来自真实用户故事，应优先保留匿名表达或授权素材。
- 小羽出现会让内容像角色美图，而不是羽毛球情绪内容。

## 经验回写机制

每次整理小红书笔记后，都要判断是否有内容值得沉淀进 `docs/yuji-company/skills/`，让下一次复用。

值得回写的内容：

- 多次可复用的笔记结构，比如“真实入坑经历 -> 成长转折 -> 情绪出口 -> 想保存的具体瞬间 -> YUJI 轻落点”。
- 明显提升真诚感的表达方式，比如先写个人具体场景，再写品牌使命。
- 新发现的品牌边界，比如第一篇不把 AI 放到前台，不写成创业求支持。
- 可复用的标题句型、封面句型、评论区提问方式。
- 发布后被评论、收藏、私信验证有效的情绪主题。

暂不回写的内容：

- 只适合单篇的个人细节。
- 未经验证、只是临时灵感的口号。
- 会把 YUJI 带偏成装备测评、技术教学、AI 炫技或低价定制的表达。

回写方式：

- 小的写作规则直接补充到本 Skill。
- 品牌语气规则补充到 `yuji-brand-voice.md`。
- 用户情绪洞察补充到 `badminton-emotion-listening.md`。
- 产品信号补充到 `product-insight.md`。
- 未使用素材补充到 `content-library/materials.md`。
- 未使用选题补充到 `content-library/topics.md`。
- 如果形成新的稳定工作流，再新增独立 Skill。

## 发布后台配置

每篇待发布笔记都要补齐后台发布建议，避免 CEO 进入小红书后台后临时判断。

## 配图文字大小规则

小红书图文不只有首图需要文字可读。第 2、3 张内容图如果承载一句情绪标题、摘录或关键转折，也必须按手机信息流可读标准写提示词。

默认规则：

- 图文第 2、3 张的文字不要写成“很小的角落注释”“小字”“旧相册注释”。
- 如果这句话承担内容转折或情绪记忆点，要写成 `醒目的中文文字`、`核心情绪标题` 或 `手机信息流预览时也能一眼读清楚`。
- 文字可以克制，但不能小到需要点开大图才能看清。
- 提示词里要明确 `字号要足够大`、`和背景有足够对比`、`可加轻微柔和阴影`。
- 保持 YUJI 的克制质感，但避免弱到没有传播效率。判断标准是：用户滑到第 2、3 张时，不看正文也能被那一句话击中。

## 羽毛球静物提示词规则

当配图包含拍包、旧拍、手胶、羽毛球、水瓶、毛巾等静物时，提示词必须写清楚这些可识别细节：

- 球拍手柄优先写成 `缠橙色手胶`，并说明 `橙色要清楚可见`。
- 羽毛球要写成 `完整度较高、羽毛排列清楚、外形可识别`。
- 避免把羽毛球写成严重破损废球，除非选题明确是战损或废球纪念。
- 旧物可以有磨损和使用痕迹，但不要脏乱、破败或像垃圾堆。
- 默认静物组合可以是：半开拍包、缠橙色手胶的旧拍、完整羽毛球、水瓶、毛巾。

### 发布类型

第一阶段默认优先 `上传图文`，尤其适合创始人故事、羽毛球情绪瞬间、旧物记忆和球友关系。

图文不是只发图片，而是：

```text
封面图
+ 标题
+ 完整正文
+ 标签
+ 发布后首评
```

封面负责让人停下，正文负责让人共鸣、评论和收藏。不要因为选择图文就删减情绪正文。

不优先选择：

- `上传视频`：除非已有真实、强情绪、能快速进入场景的视频素材。
- `写长文`：第一阶段容易像专栏文章，冷启动阅读门槛偏高。
- `发播客`：不适合当前羽毛球情绪视觉起号。

### 标签

标签先用自然场景词，让平台和用户都能理解。

优先使用：

- `#羽毛球`
- `#羽毛球日常`
- `#羽毛球爱好者`
- `#球搭子`
- `#下班去打球`
- `#最后一局`
- `#运动日常`
- `#运动记录`
- `#羽毛球人的快乐`
- `#羽毛球人的浪漫`
- `#YUJI`
- `#羽迹`

谨慎使用 `#PlayerID`、`#战损档案`、`#记忆海报`、`#羽迹时刻`。这些是 YUJI 的产品验证词，不是第一篇和纯情绪内容的默认流量标签。只有内容明确试探产品，或评论、私信已经自然出现“我也想拥有一个”后，再把它们放到前台。

### 内容类型声明和 AI

后台声明要同时考虑平台合规和品牌前台感受：

- 原创经历、原创正文和自制封面，可以开启原创声明。
- 内容类型如果需要选择，优先选运动健身、羽毛球、生活记录或日常。
- 不选商业推广、商品种草、教程科普，除非当篇内容确实如此。
- AI 生图或改图只作为幕后能力。若平台强制声明，按平台规则如实声明；若只是可选项，不主动把第一阶段内容包装成 AI 内容号。
- 正文里不要讲 AI、Prompt、生图过程或工程化。

### 组件、定时和首评

组件默认少加：

- 地点：不加，除非要测试本地球友。
- 群聊、路线、文件：不加。
- 允许正文复制：可以开启。
- 允许合拍：可开可关，对图文情绪笔记影响不大。
- 可见范围：公开可见。

定时发布期间，网页创作服务平台不一定支持提前评论，但手机端可能可以进入待发布笔记并提前添加作者评论。运营总监必须给 CEO 一条首评文案，并提醒先用手机端尝试预置首评；如果手机端可添加，正式发布后要确认评论是否同步生效；如果不能提前添加，则发布后 5 分钟内手动补首评。

推荐发布时间先测试：

- 晚间：`21:00-22:30`
- 午间：`12:00-13:00`

发布前按 `docs/yuji-company/xhs-notes/publishing-checklist.md` 做最后检查。

## 创始人故事写法

当 CEO 提供自己的真实经历时，优先把它写成“一个羽毛球人为什么想留下这些瞬间”，不要写成正式品牌公告。

可复用结构：

1. 真实入坑经历：从普通球局、菜鸟局、朋友带入门等具体场景开始。
2. 成长转折：写从快乐打球到想变强、开始早球或加入更高水平球局。
3. 情绪出口：写开心、难过、疲惫时都想去球馆，把羽毛球变成生活出口。
4. 想保存的具体瞬间：写 60 拍、杀球、凌晨球局、下班推门看到球馆灯、和搭子赢下最后一分。
5. YUJI 轻落点：落到“把热爱留下痕迹”，不硬解释产品。

表达替换：

- 不说“我有 AI 工程经验，所以我要用 AI 做这个品牌”，改成“我想用自己擅长的方式，把这些羽毛球记忆沉淀出来”。
- 不说“希望大家支持我的创业”，改成“如果你也有某个只有自己懂的羽毛球瞬间，我们应该会在这里相遇”。
- 不说“我要做一个商业项目”，改成“我想开始认真记录这些时刻”。

### CEO 改稿校准

创始人故事不要被运营总监修得过度完整、过度正确。CEO 自己改回来的口语感，往往比顺滑文案更真实。

优先保留：

- 更像本人说话的句子，例如“我第一次开始打羽毛球”，不必总改成“第一次认真开始”。
- 个人限定，例如“有时候在我心里它不是运动”，比直接下判断“它不是运动”更真诚。
- 球友关系细节，例如“推门看到球搭子等待的瞬间”，优先于只写“球馆灯亮起来”的氛围画面。
- 短句和留白，让正文像手机上慢慢打出来的心里话，而不是一篇品牌文章。
- 小红书发布正文和一键复制正文默认去掉每句末尾的中文句号。保留必要的问号、冒号、引号、省略号和句中逗号；段落之间该空行就空行，方便 CEO 直接复制到小红书后台。

改稿判断：

1. 如果 CEO 的改法让内容更口语、更具体、更像亲历者，应优先采用。
2. 如果一个画面能从“场景氛围”升级成“人与人关系”，优先写关系。
3. 不要为了文案整齐，删掉“我心里”“不知道为什么”“就是很快乐”这类真实犹豫和轻微停顿。
4. 运营总监可以修错字、节奏和不清楚的表达，但不要把创始人故事改成品牌介绍稿。

## 起步内容方向

### 羽毛球情绪瞬间

优先写普通球友立刻能代入的具体瞬间：

- 最后一局永远最久。
- 球馆灯亮起来，人就活过来了。
- 有些球不值钱，但就是舍不得扔。
- 成年人的回血方式，是下班后那两小时。
- 退役的不是球拍，是那段时间的自己。

### 羽毛球人格系列

用于提高评论和转发，语气要像球友之间互相调侃：

- 永远说“最后一局”的人。
- 输球后开始研究手胶的人。
- 打混双像打世界赛的人。
- 明明很累，但约球从不拒绝的人。
- 热身半小时，上场三分钟就喊累的人。

### 羽迹卡 / 羽迹时刻试探

只展示“这东西很酷”，不要硬卖：

- 羽迹卡故事海报。
- 球员赛季痕迹卡。
- 战损球拍档案。
- 搭子纪念卡。
- 年度羽毛球记忆海报。
- 一场球的高光与遗憾记录。

## 标题风格

标题要像球友的心里话，而不是营销口号。

示例方向：

- 最后一局，永远最久
- 有些球不值钱，但舍不得扔
- 球馆灯一亮，人就活过来了
- 退役的不是球拍，是那段时间的自己

## 封面和配图风格

首选“电影感 + 球馆灯光 + 留白 + 成年人的热爱”。

视觉关键词：

- 暖光球馆
- 空场地
- 常规羽毛球 PVC 地胶，低饱和墨绿、青绿色或蓝绿色
- 白色或黄色羽毛球场地线
- 地胶柔和反光
- 背影
- 坐在场边休息
- 系鞋带
- 旧球、旧手胶、断线球拍、磨损球鞋
- 黑色、暖白，少量金色或荧光绿

人物组合规则：

- 后续生图不要默认都画成男双。YUJI 的创始人视角是女生，常打混双，常见球搭子关系是女生和男生搭子一起打球。
- 如果选题是 CEO 个人视角、球搭子关系、新手被带、有人等你一起、混双默契，优先写混双：一位女生和一位男生球搭子。
- 如果选题明确是兄弟局、男双对抗、男生固定搭子，再写男双。
- 如果选题明确是女生球友、女生搭子、女双默契或女生运动情绪，就写女双。
- 如果选题是旧物、空球馆、战损球拍、散场静物，可以不出现人物，或只出现不露脸背影、手部、腿部动作。
- 不要把混双画成情侣写真或暧昧互动。关系感来自等球、补位、击掌、收包、讲站位和一起离场，不来自亲密摆拍。
- 女生形象要真实适合打球，穿常规羽毛球运动服或运动短袖短裤，不要网红写真、裙摆夸张、妆发摆拍或过度柔弱。
- 小羽在不同笔记中可以更换服装，优先从 `docs/yuji-company/visual-system/xiaoyu/outfit-library.md` 选择最近 14 天未使用过的一套；但同一篇多图人物必须保持统一造型。

封面成品建议加短文字，是否在生图时直接生成文字，由当次发布目标决定。

封面文字规则：

- 主封面通常需要 6-12 字短文案，帮助信息流里快速识别主题。
- 文字应像球友心里话，不像营销标题。
- 如果 ChatGPT 生图能稳定处理文字，可以直接在提示词里要求生成封面文字，并明确文字内容、位置、字体气质和留白。
- 如果生成文字错字、糊字或排版不稳，再改用干净画面后期手动叠字。
- 如果画面本身信息极强，可以做无字封面测试，但 YUJI 起号阶段优先用“情绪画面 + 短文字”。
- 字体和排版要克制，避免淘宝风大字报、鸡血标语、低龄贴纸和复杂角标。
- 所有带字生图提示词都必须包含这句字体设计要求：`字体参考运动品牌 YUJI Logo 的干刷墨迹质感：粗糙笔触、轻微飞白、边缘有纸感颗粒和沙沙的磨损感，但不要太重，不要像正式书法作品。`
- 怀旧向内容尤其要避免细宋体、端庄书法体、古风字体；封面字要比 Logo 更轻、更清楚，不能喧宾夺主。

怀旧向封面文字要带“时间感”，不能只写物件本身。优先使用：

- 时间钩子：`那天`、`那年`、`最后一局`、`打完那晚`。
- 留存动作：`还在拍包里`、`一直没舍得扔`、`留着的是那天`。
- 轻微遗憾：`那局回不去了`、`它也在场`。
- 人对物说话：`我还把你留在包里`、`我也不舍得扔你`、`你也在场`。

避免只写“旧球 / 你还留着吗”这类单纯提问。它能表达主题，但怀旧感不够。

旧物怀旧主题优先使用第二人称口吻，把旧球、旧拍、旧手胶当成陪过人的对象来写。这样更像人在回看一段时间，而不是运营在描述一个物件。

封面文案尽量一句话，不堆字，不做淘宝风、鸡血风、AI 炫技风或低龄动漫风。

每次给出配图建议时，必须同时给出对应的 ChatGPT 生图提示词。提示词要包含：

- 每条提示词都必须完整、独立、可直接复制使用，不能依赖前文说明或上一条提示词。
- 小羽提示词优先从 `outfit-library.md` 选择一套服装；不同笔记可以更换服装，优先最近 14 天不重复。
- 同一篇多图人物必须保持统一造型。每条提示词都要重复写明同一套服装和道具，例如 `浅灰短袖运动 T 恤、黑色运动裙裤、白色羽毛球鞋、白色短袜、黑色拍包、低马尾`；不能只在第一条里写，也不能只写“保持一致”。
- 羽迹工作室默认不由 Codex 直接生成正式小红书图片。除非 CEO 明确要求“你直接生成图片”，否则只给参考图方向和可复制到 ChatGPT 的独立生图提示词，由 CEO 自己生成、挑图和回传验收。
- 如果需要参考图，优先给“参考图应该长什么样”的文字说明，或引用项目内已有样稿路径；不要把参考图当作最终发布图。
- 画幅：优先竖版 `3:4`，适合小红书。
- 风格：写实摄影、电影感、真实运动生活方式。
- 主体：旧球、旧拍、旧手胶、球馆、背影等具体物件或场景。
- 场地：必须写清楚常规羽毛球 PVC 地胶，颜色为低饱和墨绿、青绿色或蓝绿色；场地线为白色或黄色；可以有地胶柔和反光、汗印和灯光倒影；不要写木地板、篮球馆、排球馆、红色跑道、灰色水泥地或综合体育馆地面。
- 人物组合：根据场景明确写混双、男双、女双或无人静物。默认不要一味男双；CEO 个人视角、球搭子关系和被男生搭子带着打的场景，优先写一位女生和一位男生球搭子。
- 光线：暖光球馆、地胶柔和反光、低饱和、克制留白。
- 构图：近景、低角度、虚化背景、画面留白等。
- 情绪：下班后、散场后、最后一局、疲惫但留恋。
- 文字：如果需要封面字，明确写出 exact text、位置和字体气质，并必须包含 YUJI Logo 干刷墨迹字体标准句；如果不需要，明确写“不要生成文字”。
- 禁止项：不要 logo、不要商品广告感、不要夸张光效、不要清晰人脸。

提示词只作为幕后生图工具使用，前台笔记正文不要提 AI 或生图过程。

### 成图验收

CEO 生成图片后，运营总监要先做视觉验收，再进入发布：

- 文件：确认图片已复制到项目目录，例如 `ready-to-publish/assets/YYYY-MM-DD-序号/`，不要只依赖桌面路径。
- 尺寸：优先 3:4 竖图，记录实际尺寸。
- 场地：检查是否为常规羽毛球 PVC 地胶，颜色和场地线是否正确。
- 人物：检查人物组合是否符合当篇场景，后续默认不要一味男双。
- 文字：检查封面和内容图文字是否无错字、可读、不遮挡主体。
- 叙事：检查三张图是否能按顺序承接主题，不只是三张好看的图。
- 稿件：通过后在待发布稿写入 `配图定稿` 和 `视觉审核`。

## 互动观察

发布后重点记录以下信号：

- 评论里是否出现“这就是我”“太真实了”“想起我的搭子”。
- 是否有人收藏或转发给球友。
- 是否有人私信问 Player ID、战损档案、记忆海报能不能做。
- 是否有人主动讲自己的旧球、旧拍、搭子、最后一局故事。
